Дата исследования: Март 2026
Модели: Elphapex DG1, DG1+ (Прошивки на базе OpenWrt/LuCI)

1. ГЛАВНЫЙ СЕКРЕТ (ОБХОД ПАРОЛЯ)
---------------------------------------------------------
Официальная программа (Elphapex Tool) НЕ использует порт 9588 для управления. 
Она работает по стандартному порту 80 (HTTP), но стучится в скрытую директорию `/cgi-bin/luci/`.
Запросы в эту директорию обрабатываются МГНОВЕННО и НЕ ТРЕБУЮТ АВТОРИЗАЦИИ (обход root:root).

2. ЭНДПОИНТЫ СТАТИСТИКИ (GET запросы, без пароля)
---------------------------------------------------------
* http://{ip}/cgi-bin/luci/stats.cgi          - Вся статистика по хешу и платам.
* http://{ip}/cgi-bin/luci/get_miner_conf.cgi - Конфиг пулов и частот.
* http://{ip}/cgi-bin/luci/get_system_info.cgi- Версия прошивки, MAC, аптайм.
* http://{ip}/cgi-bin/luci/getworkmode.cgi    - Текущий режим (0 = норма, -1000 = сон).

3. КОМАНДЫ УПРАВЛЕНИЯ (REMOTE CONTROL)
---------------------------------------------------------
[!] Выполняются без пароля напрямую в /luci/

А) ПОИСК АСИКА (Моргание лампочкой)
* Метод: POST
* URL: http://{ip}/cgi-bin/luci/ftm_ledtest.cgi
* Payload (ВКЛ):  {"leds_blue": 1, "leds_red": 0, "leds_flash": 1, "leds_time": 0}
* Payload (ВЫКЛ): {"leds_blue": 0, "leds_red": 0, "leds_flash": 0, "leds_time": 0}
* Ответ успеха: {"stats":"success","msg":"OK!","code":"M000"}

Б) РЕЖИМ СНА (Sleep / Wake up)
* Метод: POST
* URL: http://{ip}/cgi-bin/luci/setworkmode.cgi
* Payload (УСНУТЬ): {"workmode": "-1000"}
* Payload (ПРОСНУТЬСЯ): {"workmode": "0"}

В) ПЕРЕЗАГРУЗКА (Reboot)
* Метод: GET (!)
* URL: http://{ip}/cgi-bin/luci/reboot.cgi
* Примечание: Асик просто рвет соединение и уходит в ребут.

4. ОСОБЕННОСТИ МАТЕМАТИКИ ХЕШРЕЙТА ELPHAPEX
---------------------------------------------------------
* Блока "SUMMARY" в stats.cgi нет.
* Параметр `rate_5s` в корне блока STATS сломан (выдает космический мусор вроде 122552.17).
* ПРАВИЛЬНЫЙ хешрейт (Real HR) нужно высчитывать вручную, суммируя параметр `hashrate` (или `rate_real`) из массива каждой платы `chain`.
* Значения отдаются строго в MH/s (для алгоритма Scrypt их нужно делить на 1000 для перевода в GH/s).
