# Canaan Avalon API Reference

**Тип API:** Avalon JSON API
**Особенность:** Скрытые данные в строках

## 1. Идентификация модели
Модель асика можно извлечь двумя путями:
1. Из блока `VERSION` -> `PROD`.
2. Из скрытого массива данных (если стандартное поле пустое).

## 2. "Секретный" блок MM ID
Avalon упаковывает часть системной информации и конфигураций внутрь длинных текстовых строк с ключом `MM ID`.
**Метод парсинга:** Используется регулярное выражение `r'(\w+)\[([^\]]*)\]'` для извлечения пар "ключ-значение" из этой строки (например, скрытые температуры или версии чипов).

## 3. Сбор метрик
* **Хешрейт:** Находится в ключах `GHS mm` (средний) и `GHS 5s` (текущий). Значение отдается в GH/s.
* **Вентиляторы:** Записаны в плоском словаре как `Fan1`, `Fan2` ... `Fan8`.
* **Температура:** Берется из `Temp`, если значений много, программа оставляет только крайние (мин/макс).
