# Antminer (Stock Firmware) API Reference

**Основа:** CGMiner / BMiner API
**Главные порты:** `4028` (TCP API), `80` (Web/CGI), `6060` (Errors)

## 1. Основной API (Порт 4028)
Быстрый TCP-сокет. Принимает текстовые JSON-команды.
* **Команды:** `{"command": "summary"}`, `{"command": "stats"}`, `{"command": "pools"}`.
* **Сон:** Если `Elapsed` (аптайм) > 60 сек, а хешрейт = 0 (и кулеры = 0), устройство находится в режиме сна.

## 2. Веб-интерфейс (Порт 80)
Используется как резервный канал, если порт 4028 завис или закрыт (например, во время сна асика).
* Требует HTTP Digest / Basic Auth (обычно `root:root`).
* `http://{ip}/cgi-bin/get_system_info.cgi`
* `http://{ip}/cgi-bin/get_miner_conf.cgi`

## 3. Мониторинг аппаратных ошибок (Порт 6060)
* `http://{ip}:6060/warning`
* Отдает текстовую строку с кодами ошибок (например, отвалившиеся хешплаты, проблемы с сетью или кулерами). Если порт отвечает — приоритет отдается ошибке, а не статусу "Сон".

## 4. Защита от спуфинга (Ложные Antminer)
Многие сторонние производители (Hammer, Bluestar, iPollo, Jasminer, Canaan) используют порт 4028 и маскируются под Antminer.
**Методы жесткой фильтрации:**
1. На порту 4028 (в дампе `STATS` или `VERSION`) ищем запрещенные слова: `hammer`, `bluestar`, `canaan`, `jasminer`, `ipollo`.
2. На порту 80 жестко проверяем `hostname` и `minertype`. Если нет слова `Antminer` или `Bitmain`, но есть `Bluestar` / `Hammer` — отбрасываем.
