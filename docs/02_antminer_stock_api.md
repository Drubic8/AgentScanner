# Antminer Stock API Reference (Заводская прошивка)

**Основной порт управления:** `80` (HTTP CGI-скрипты)
**Порт диагностики (контрольная плата):** `6060`
**Авторизация:** HTTP Digest Auth (По умолчанию логин/пароль: `root`:`root`)

В отличие от кастомных прошивок (Vnish, Braiins), стоковая прошивка Bitmain не имеет единого REST API. Управление происходит через имитацию действий пользователя в веб-интерфейсе (вызов `.cgi` скриптов).

## 1. Управление подсветкой (Поиск асика)
В новых версиях стоковой прошивки скрипт управления LED строго требует передачи данных в формате **JSON**.
* **Эндпоинт:** `POST /cgi-bin/blink.cgi`
* **Заголовки:** `Content-Type: application/json`
* **Payload (Включить):** `{"blink": true}`
* **Payload (Выключить):** `{"blink": false}`
* **Успешный ответ:** `{"code":"B000"}`

## 2. Перезагрузка (Reboot)
Самый надежный способ перезагрузить стоковый Antminer с корректным сохранением логов и остановкой процессов.
* **Эндпоинт:** `GET /cgi-bin/reboot.cgi`
* **Успешный ответ:** HTTP 200 OK. Асик уйдет в ребут через 5-15 секунд.

## 3. Режим работы (Сон / Майнинг)
У Bitmain нет отдельной кнопки "Сон" (Suspend). Перевод устройства в спящий режим осуществляется путем изменения конфигурации асика (`work_mode`).
**Важно:** Нельзя отправлять только статус работы, иначе затрутся настройки пулов. Правильный алгоритм:
1. **Получить текущий конфиг:** `GET /cgi-bin/get_miner_conf.cgi`
2. **Изменить ключ:** В полученном JSON найти `"bitmain-work-mode"` и установить `"1"` (Сон) или `"0"` (Нормальная работа).
3. **Отправить обратно:** `POST /cgi-bin/set_miner_conf.cgi` с обновленным JSON в теле запроса.

## 4. Чтение ошибок (Порт 6060)
Диагностический порт 6060 работает без пароля и отдает системные предупреждения контроллера напрямую.
* **Эндпоинт:** `GET http://<IP>:6060/warning`
* **Формат ответа:** Текст, разделенный точкой с запятой (`;`). 
* **Пример ответа:** `W040;Fan speed is too high;Please check...`

## 4. Определение статуса Сна (Sleep Mode) на новых прошивках
Исторически для определения спящего режима использовался параметр `Mode: 1` или опрос скрытого API на порту `6060`. Однако новые поколения асиков (L9, T21, S21, KS5) изменили это поведение:
* **Ключ Mode:** На новых прошивках значение `Mode: 2` означает **нормальный рабочий режим** (Performance/Normal), а не сон. Использование этого ключа для детекции сна приведет к ложным срабатываниям.
* **Порт 6060:** Скрытый HTTP-сервер на порту 6060 (эндпоинты `/miner_status`, `/readvol`) работает крайне нестабильно, часто обрывает соединение или возвращает `404 Not Found`. Мы используем его **только** для получения кодов аппаратных ошибок (`/warning`).

**Золотое правило детекции сна (Порт 4028):**
Самый надежный способ определить, что стоковый Antminer находится в режиме сна — это проверить физические показатели через команду `stats` (CGMiner API). Асик гарантированно спит, если выполняются **все три условия одновременно**:
1. `Hashrate == 0`
2. `fan_num == 0` (Кулеры остановлены)
3. `temp_max == 0` (Датчики температуры отключены)
