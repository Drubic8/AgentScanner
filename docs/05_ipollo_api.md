# iPollo API Reference

**Тип API:** CGMiner/BMiner JSON API (Порт 4028)
**Алгоритмы:** Ethash, Cuckatoo31 (MWC), Cuckatoo32 (GRIN)

## 1. Особенности протокола
iPollo использует классический API-интерфейс, однако имеет нестандартные обозначения моделей и алгоритмов внутри дампа `STATS`.

## 2. Кастомный маппинг моделей
Аппараты iPollo отдают внутренние заводские индексы, которые нужно преобразовывать в коммерческие названия (в поле `ID`):
* `G220` -> iPollo G1
* `G1` -> iPollo G1 Mini
* `V1` -> iPollo V1 Mini
* `X1` -> iPollo X1
* `B1` -> iPollo B1

## 3. Маппинг алгоритмов
Сырые данные алгоритма (`Algo`) также требуют перевода:
* `mwc` -> Cuckatoo31 (MWC)
* `grin` -> Cuckatoo32 (GRIN)

## 4. Сбор метрик
* **Хешрейт:** Отдается в G/s (для MWC/GRIN) или в MH/s. Для внутренних нужд дашборда G/s делится на 1000 и сохраняется как TH/s.
* **Температуры:** В поле `Temp` выдается строка с множеством значений. Программа извлекает все числа и берет только первое (минимальное) и последнее (максимальное) значение.
