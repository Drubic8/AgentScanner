# Jasminer API Reference

**Тип API:** Custom JSON API
**Алгоритм:** ETCHASH / ETHASH

## 1. Архитектура ответа
Jasminer разбивает данные на три логических блока в едином JSON-ответе:
* `summary` — Модель (`miner`), аптайм (`uptime`), хешрейты (`rt`, `avg`).
* `pools` — Конфигурация пулов.
* `boards` — Статистика хешплат (кулеры, температуры).

## 2. Определение активного пула
В отличие от других асиков, Jasminer может отдавать большой список пулов, где нужно искать активный по статусу. 
Рабочий пул имеет статус `in use` или `alive`. Если таких нет, фоллбэк идет на `pool: 0`.

## 3. Хешплаты и охлаждение
* Вентиляторы извлекаются из блока `boards` -> `fans` (скорость в RPM).
* Температуры (`temp`) представлены массивом. Для корректного отображения берется максимальная температура среди всех плат.
