# Jasminer API Reference

**Тип API:** Custom HTTP Web/JSON API (Порт 80)
**Алгоритм:** ETCHASH / ETHASH
**Авторизация:** HTTP Digest Auth (по умолчанию `root:root`)

## 1. Основные эндпоинты (Data Endpoints)
В отличие от большинства ASIC-майнеров, использующих сокеты на порту 4028, Jasminer отдает статистику через веб-сервер (lighttpd). 

Для мгновенного получения полных данных (без каскада таймаутов) используется скрытый эндпоинт:
* `GET /cgi-bin/minerStatus_all.cgi` — Возвращает полный JSON со статусом устройства, минуя долгие проверки интерфейса.

*(Дополнительные эндпоинты, используемые официальным софтом: `/cgi-bin/index.cgi`, `/cgi-bin/minerStatus.cgi`, `/cgi-bin/get_pools.cgi`)*.

## 2. Команды управления (Control Commands)
Управление устройством осуществляется через отправку `POST` запросов с Digest-авторизацией.

* **Поиск майнера (LED Blink):**
  * **Запрос:** `POST /cgi-bin/find_miner_on.cgi`
  * **Описание:** Включает мигание светодиода на корпусе для физического поиска устройства на стеллаже.
  * **Ответ:** Возвращает строку `ok`.

## 3. Архитектура ответа (`minerStatus_all.cgi`)
Jasminer разбивает данные на три логических блока в едином JSON-ответе:
* `summary` — Модель (`miner`), аптайм (`uptime`), хешрейты (`rt`, `avg`).
* `pools` — Конфигурация пулов.
* `boards` — Статистика хешплат (кулеры, температуры).

## 4. Определение активного пула
В отличие от других асиков, Jasminer может отдавать большой список пулов, где нужно искать активный по статусу. 
Рабочий пул имеет статус `in use` или `alive`. Если таких нет, фоллбэк идет на `pool: 0`.

## 5. Хешплаты и охлаждение
* Вентиляторы извлекаются из блока `boards` -> `fans` (скорость в RPM).
* Температуры (`temp`) представлены массивом. Для корректного отображения берется максимальная температура среди всех плат.
