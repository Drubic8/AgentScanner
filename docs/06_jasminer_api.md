# Jasminer API Reference

**Тип API:** Custom HTTP Web/JSON API (Порт 80)
**Алгоритм:** ETCHASH / ETHASH
**Авторизация:** HTTP Digest Auth (по умолчанию `root:root`)

## 🚨 Важная особенность (Сетевой протокол)
В отличие от большинства ASIC-майнеров, использующих сокеты на порту 4028, Jasminer отдает статистику через встроенный веб-сервер (lighttpd). 

**КРИТИЧЕСКОЕ ПРАВИЛО:** * Jasminer строго ожидает **`POST`**-запросы. Использование `GET`-запросов может приводить к игнорированию пакетов со стороны сервера и жестким таймаутам (задержки сканирования до 30+ секунд).
* При отправке запросов с Digest-авторизацией **ОБЯЗАТЕЛЬНО** использовать сессии (`Keep-Alive`), чтобы избежать двойного TCP-рукопожатия на каждом запросе, иначе время отклика вырастет в 2-3 раза.

## 1. Основные эндпоинты (Data Endpoints)

Для мгновенного получения абсолютно всех данных об устройстве (включая хешрейт, кулеры, температуры и пулы) используется один универсальный эндпоинт:

* **Запрос:** `POST /cgi-bin/minerStatus.cgi`
* **Описание:** Возвращает полный JSON со статусом устройства. Отрабатывает за 0.5 - 2 секунды.

*(Дополнительные эндпоинты, используемые официальным софтом: `/cgi-bin/index.cgi` (только модель и аптайм), `/cgi-bin/get_pools.cgi` (только пулы), `/cgi-bin/minerStatus_all.cgi` (устаревший/отсутствует на X16-Q))*

## 2. Команды управления (Control Commands)
Управление устройством осуществляется через отправку `POST` запросов с Digest-авторизацией.

* **Поиск майнера (LED Blink):**
  * **Запрос:** `POST /cgi-bin/find_miner_on.cgi`
  * **Описание:** Включает мигание светодиода на корпусе для физического поиска устройства на стеллаже.
  * **Ответ:** Возвращает строку `ok`.

## 3. Архитектура ответа (`minerStatus.cgi`)
Начиная с новых прошивок (например, X16-Q), единый ответ от `minerStatus.cgi` включает в себя корневые ключи:
* `summary` — Модель (`miner`), аптайм (`uptime`), текущий хешрейт (`rt`), средний хешрейт (`avg`). *Примечание: хешрейт может отдаваться строкой вместе с единицами измерения, например `"2214.48 MH/s"`*.
* `pools` — Конфигурация пулов.
* `boards` — Статистика хешплат (вентиляторы `fan1...fan6` и детальные температуры по чипам).

## 4. Определение активного пула
В отличие от других асиков, Jasminer может отдавать большой список пулов, где нужно искать активный по статусу. 
Рабочий пул имеет статус `in use`, `alive` или `active`.
