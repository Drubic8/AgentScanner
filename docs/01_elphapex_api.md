# Elphapex API Reference (Reverse Engineered)

**Модели:** DG1, DG1+ (Прошивки на базе OpenWrt/LuCI)
**Исследовано:** Март 2026 (Сниффинг трафика Elphapex Tool)

## 1. Главный секрет (Обход пароля и скорости)
Официальная утилита не использует порт 9588 для сбора статистики. Вся работа идет по стандартному **Port 80 (HTTP)**, но через скрытую директорию `/cgi-bin/luci/`.
**Особенность:** Запросы в `/luci/` обрабатываются мгновенно и **НЕ ТРЕБУЮТ АВТОРИЗАЦИИ** (обходят root:root).

## 2. Эндпоинты Статистики (GET-запросы)
* `http://{ip}/cgi-bin/luci/stats.cgi` - Статистика хешрейта, температур, оборотов кулеров.
* `http://{ip}/cgi-bin/luci/get_miner_conf.cgi` - Конфигурация пулов и воркеров.
* `http://{ip}/cgi-bin/luci/get_system_info.cgi` - MAC, аптайм, версия прошивки.
* `http://{ip}/cgi-bin/luci/getworkmode.cgi` - Текущий статус работы (`0` = норма, `-1000` = сон).

## 3. Управление устройствами (Remote Control)
Все команды отправляются в директорию `/luci/` без пароля.

* **Мигание LED (Поиск):** * Метод: `POST` на `/cgi-bin/luci/ftm_ledtest.cgi`
  * Включить: `{"leds_blue": 1, "leds_red": 0, "leds_flash": 1, "leds_time": 0}`
  * Выключить: `{"leds_blue": 0, "leds_red": 0, "leds_flash": 0, "leds_time": 0}`
* **Сон (Standby):**
  * Метод: `POST` на `/cgi-bin/luci/setworkmode.cgi`
  * Payload: `{"workmode": "-1000"}` (Уснуть) / `{"workmode": "0"}` (Проснуться)
* **Перезагрузка (Reboot):**
  * Метод: `GET` на `/cgi-bin/luci/reboot.cgi` (Асик моментально обрывает соединение).

## 4. Особенности хешрейта
* Параметр `rate_5s` внутри `STATS` выдает мусор (ошибочные гигантские значения).
* **Правильный хешрейт** высчитывается путем сложения параметра `hashrate` (или `rate_real`) со всех элементов массива `chain`.
* Единица измерения отдаваемых данных — **MH/s** (для Scrypt делим на 1000 для перевода в GH/s).
